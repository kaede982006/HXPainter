Latest Wine verification notes

The current Windows package was tested from a Linux host through Wine after the canvas was moved from QOpenGLWidget to a QWidget-backed renderer.

Verified commands:
- ./manage.sh win
- ./manage.sh test-win
- wine HXPainter.exe --startup-smoke-test
- timeout 6s wine HXPainter.exe

Expected startup behavior:
- --startup-smoke-test exits with code 0 after confirming that the main window is visible and the default document is created.
- Running HXPainter.exe without arguments stays alive until the user closes the window. In automated checks this appears as timeout exit code 124 when wrapped with timeout 6s.

Packaging notes:
- The Windows runtime archive includes HXPainter.exe, logo.png, Qt6Core.dll, Qt6Gui.dll, Qt6Widgets.dll, Qt6Svg.dll, MinGW runtime DLLs, and Qt plugin directories such as platforms, imageformats, iconengines, and styles.
- The current Windows build does not link or ship Qt6OpenGL.dll, Qt6OpenGLWidgets.dll, opengl32sw.dll, or libgallium_wgl.dll.
